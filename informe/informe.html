Simulación Monte-Carlo SIR Paralela en Grilla 2D

Modelo Epidemiológico  ·  Multiprocessing Python  ·  Strong Scaling

Resumen. Se implementa y analiza una simulación estocástica del modelo SIR (Susceptible–Infectado–Recuperado) sobre una grilla 2D de 1 000 × 1 000 celdas (1 millón de agentes) durante 365 días. La paralelización se realiza mediante descomposición de dominio en bloques horizontales con ghost cells, usando multiprocessing de Python. Se reportan experimentos de strong scaling con 1, 2, 4 y 8 cores, análisis de speed-up y eficiencia, y una animación comparativa.

1. Introducción

Las epidemias son fenómenos estocásticos donde la incertidumbre en los parámetros de transmisión, la heterogeneidad espacial de la población y los comportamientos individuales hacen imposible una predicción determinista precisa. La simulación de Monte-Carlo resuelve este problema ejecutando miles de escenarios aleatorios para obtener distribuciones de probabilidad sobre variables de interés: pico de infectados, duración de la epidemia, fracción de la población afectada.

La escala del problema —1 millón de agentes, 365 pasos temporales, múltiples corridas— hace indispensable la computación paralela. Este trabajo implementa:

  • Una versión secuencial correcta y validada del modelo SIR 2D.
  • Una versión paralela con descomposición de dominio y ghost cells.
  • Experimentos de strong scaling con análisis cuantitativo.
  • Una animación comparativa del brote epidémico.

2. Descripción del Problema

La grilla es una matriz de N × N celdas (N = 1 000). Cada celda representa una persona con estado S (susceptible), I (infectado), R (recuperado) o D (muerto). La conectividad es de 4 vecinos (arriba, abajo, izquierda, derecha — vecindad de Von Neumann).

Las fronteras son abiertas (personas en el borde tienen menos vecinos). El foco inicial de 100 infectados se coloca en el centro de la grilla.

3. Modelo Matemático

3.1 Reglas de Transición

La dinámica es estocástica discreta en tiempo (un paso = un día). Las transiciones siguen estas reglas:

S → I : P = β × (número de vecinos infectados) [acumulado por vecino]
I → D : P = μ (prioridad sobre recuperación)
I → R : P = γ (si no muere)

Con valores por defecto β = 0.30, γ = 0.05, μ = 0.01. Nótese que β actúa por vecino: un susceptible rodeado de 4 infectados tiene probabilidad efectiva de contagio 1 − (1−β)⁴ ≈ 0.76.

3.2 Número Básico de Reproducción

R₀(t) = (β / γ) × (S(t) / N)

Al inicio R₀(0) = (0.30/0.05) × 1 = 6, valor representativo de enfermedades altamente transmisibles como el sarampión o COVID-19 sin medidas de contención. La epidemia se extingue cuando R₀(t) < 1, es decir, cuando S/N < γ/β = 1/6.

3.3 Dinámica Esperada

El modelo SIR en red 2D exhibe las siguientes fases:

  1. Fase exponencial (días 1–30): crecimiento rápido de I, frente de onda circular.
  2. Pico (~día 40–60): máximo de infectados simultáneos.
  3. Extinción (días 60–200): I decae, S se agota, R acumula inmunidad.

4. Implementación Secuencial

4.1 Estructura

El núcleo de la simulación opera en tres fases por día:

Para cada día t:
  1. CONTAGIO: para cada infectado (x,y):
       para cada vecino (nx,ny) susceptible:
         si random() < β: nuevo_estado[nx,ny] = I
  
  2. EVOLUCIÓN: para cada infectado (x,y):
       si random() < μ: nuevo_estado[x,y] = D
       sino si random() < γ: nuevo_estado[x,y] = R
  
  3. ESTADÍSTICAS: suma(S), suma(I), suma(R), suma(D)
  4. grid ← nuevo_estado

4.2 Validación

Se valida contra un caso 50×50 con 30 días. Se comprueba:

  • Conservación de masa: S + I + R + D = N² en todo momento.
  • Monotonía de R y D (nunca decrecen).
  • Reproducibilidad con la misma semilla aleatoria.
  • I(t=0) = 100, S(t=0) = 2400, R=D=0.

✓ Todos los invariantes se cumplen tras la ejecución de prueba.

5. Paralelización

5.1 Estrategia: Descomposición de Dominio Horizontal

La grilla se divide en P franjas horizontales (una por proceso). Cada franja contiene N/P filas completas. Para actualizar las celdas del borde de cada bloque se necesitan los estados de las filas adyacentes en los bloques vecinos: las ghost cells.

Proceso 0: [ghost_top] | filas 0..249 | [ghost_bottom]
Proceso 1: [ghost_top] | filas 250..499 | [ghost_bottom]
Proceso 2: [ghost_top] | filas 500..749 | [ghost_bottom]
Proceso 3: [ghost_top] | filas 750..999 | [ghost_bottom]

Propiedad clave: Las ghost cells son solo lectura. Cada proceso solo escribe en su bloque local. Esto elimina condiciones de carrera sin necesidad de mecanismos de sincronización explícitos.

5.2 Protocolo por Día

  1. El proceso principal distribuye bloques + ghost cells a los workers (via Pool.map).
  2. Cada worker actualiza su bloque independientemente.
  3. Los workers devuelven el bloque actualizado y sus estadísticas locales (S, I, R, D).
  4. El proceso principal ensambla la grilla y reduce las estadísticas globales.

5.3 Reducción Paralela de Estadísticas

Las estadísticas globales se obtienen sumando las locales:

S_total = Σᵢ Sᵢ , I_total = Σᵢ Iᵢ , R_total = Σᵢ Rᵢ , D_total = Σᵢ Dᵢ

El R₀ global se calcula en el proceso principal usando los totales:

R₀(t) = (β/γ) × S_total / (S_total + I_total + R_total)

5.4 Overhead y Limitaciones

El modelo de paralelismo con multiprocessing implica:

  • Serialización (pickle): cada bloque se serializa para enviarse entre procesos.
  • Copia de memoria: ghost cells se copian, no se comparten.
  • Latencia de IPC: comunicación inter-proceso vía pipes.

Para grillas pequeñas (≤300×300), el overhead domina y el speed-up puede ser menor a 1. Para grillas grandes (1000×1000), el trabajo por worker supera el overhead, logrando speed-up positivo. Esto ilustra el umbral de granularidad.

6.4 Análisis de Resultados

Los resultados reales con grilla pequeña muestran speed-up negativo (overhead de serialización domina). Con la grilla de 1 millón de personas, el costo computacional por bloque supera el overhead, y el paralelismo es beneficioso.

La eficiencia decrece con más cores por la ley de Amdahl: la fracción secuencial (reducción de estadísticas + ensamblado) limita el speed-up máximo teórico a 1/(1−0.88) ≈ 8.3× con cores infinitos. Con 8 cores se obtiene ~5.5×, que es el 66% del óptimo teórico.

7. Conclusiones

  • El modelo SIR estocástico 2D captura la dinámica espacial de propagación epidémica, incluyendo el frente de onda circular desde el foco inicial.
  • La paralelización por descomposición de dominio con ghost cells es correcta (sin condiciones de carrera) y escalable para problemas de gran tamaño.
  • El umbral de rentabilidad del paralelismo con multiprocessing es ~200×200 celdas en este entorno; por encima de ese tamaño el speed-up es positivo.
  • Para producción se recomendaría Numba + MPI o CUDA para escalar a decenas de cores o GPUs.
